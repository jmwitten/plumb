# Cutting Guide v1 — homeowner-first fabrication surface (DB40)

**Arc brief:** SESSION UPDATE 16 NEXT ARC + owner homeowner-first ruling
(2026-07-14, binding). Branch: `fable/instruction-grammar-v1` (the consumer
surface ships separately until owner acceptance; this extends that surface).

## What ships

`frameless_three_drawer_40_cutting_guide.html` — the primary preparation
path for the builder who buys sheet goods and hardware and cuts the wood
himself with a tape measure. The mm-exact fabrication packet survives as
the opt-in shop alternate; for future builds it is NOT generated by default
(owner addendum). Once this guide passes its own naive-reader loop, the
consumer manual's kit gate is repointed here.

## Sources (all typed, all live — nothing hand-typed)

- `project.artifacts.fabrication_steps` — the 13 released `fab.*`
  WorkSteps (phases 10–100). Frame captions restate them in the homeowner
  register; every numeric token must be in `allowed_numbers` (the caption
  contract re-audits).
- `project.artifacts.cut_list` — CutListItem rows (pre-band blanks) feed
  the per-thickness buy/cut card in the tape register
  (`_reader_len`-style: clean sixteenth → fraction; else ≈fraction + exact
  mm).
- `project.model.machining` — MachiningFeature rows are the only source
  for groove/boring diagrams. The toe/attachment plan-diagram pattern
  generalizes: one part-plan builder (true-proportion rect from the part's
  cut-list L×W, groove bands as rects, bores as circles at
  location+pitch×count, datum labels from `coordinate_system`), every mark
  carrying `model_point_mm` + `fact_ref`.
- `project.artifacts.edge_banding` — banding step content.
- Release record: same per-part material/face/grain/nesting/approval
  fields as the fabrication packet's purchasing-and-cutting release record
  (`record_fields` on the final frame → record page).

## Structure (reuses the consumer layer unchanged)

A lightweight fab panel manual (panels grouped from the 13 steps,
`source_events=()` — fab steps carry no CPG placement events; ownership
contract holds trivially) → `FrameSpec`s → `project_action_frames` →
`compose_consumer_manual` → `render_consumer_manual_html`.

- Cover: cabinet render; caption states this document's job and that
  assembly continues in the consumer manual.
- Kit/inventory page: per-thickness cut lists, tape register, qty × reader
  name — L × W. No hardware letters (hardware buying stays on the consumer
  manual's kit card); `letters=()`.
- Frames: one per fab work step (some split/merged only where a single
  builder action demands it). Groove/boring frames embed typed
  OperationDiagrams from machining rows; scene images render with the
  frame's affected parts in focus, `high_contrast`.
- Dense coordinate lists stay exact mm (layout data, per the owner rule);
  single tape-markable lengths use the reader register.
- No hold gate (nothing installation-gated in fabrication); final frame =
  labeling + release record → record page ends the document.
- Machine/catalog tokens (part ids, SKUs, system ids) forbidden in all
  frame text — tools phrased as "the maker's …" like the consumer manual.

## Honesty limits (stated in the doc where they apply)

- Sheet nesting/kerf/yield are not modeled — the guide never claims a
  sheet count.
- Pilot depths and tool feeds the sources don't specify stay unclaimed.
- The fabrication packet remains the mm-exact authority; the guide links
  to it as the shop alternate.

## Acceptance

- Caps: ≤12 printed Letter pages (Chrome-verified N=N), ≤1500 visible
  instructional words.
- Mutation tests: cut-list dimension → kit card moves; machining count →
  caption/diagram moves; a stale hand-written count fails the build;
  per-thickness grouping follows mutated thickness.
- Every fab step id appears in exactly one frame's `source_step_ids`
  (completeness over the 13 released steps — machine-checked).
- Adversarial review (fresh agent) then naive-reader loop (fresh
  no-context reader per round) until a reader can reconstruct the cutting
  work correctly.
- Kit-gate repoint of the consumer manual happens only after the loop
  passes; consumer manual page count re-verified after.
