# DB40 consumer manual integration plan

## Goal

Ship the DB40 as a reciprocal pair: the existing technical/evidence build
document and a separate illustrated, self-contained assembly manual. Both use
the same compiled project, canonical reader names, and validated CPG order.

## Platform changes

1. Generalize `InstructionManual` metadata so the shared renderer no longer
   hard-codes caddy title, five-panel wording, or hot-liquid proof limits.
2. Allow an explicit set of existing site-context parts to be excluded from a
   consumer manual while retaining loud completeness for every built part.
3. Refine drawer-base placement stages into truthful milestones that preserve
   the captured-back opening/closure order and keep independently buildable
   drawer boxes in one batch stage.
4. Add a cabinetry adapter that enriches the CPG panels with model-backed
   artifact instructions, sourced hardware counts, shop tools, and the pack's
   honest structural/procurement boundary.
5. Generate the technical document and manual from one `PackedProject`, embed
   content-keyed panel images, add reciprocal relative links, and feed the same
   panel schedule to the technical document's 3D step slider.

## Acceptance

- Every non-site compiled part appears in exactly one first-arrival panel.
- The manual's panel order is the canonical CPG reader order; presentation adds
  no order edges and does not move any validation verdict.
- Captured back arrives while the right side and rear stretcher are still open.
- Hardware inventory includes physical quantities and procurement meaning from
  the artifact model, never hand-copied catalog literals.
- Technical and manual HTML are offline/self-contained and reciprocal-linked.
- A browser check finds no console warnings/errors and a fresh handyman review
  finds no unresolved Critical build-comprehension defect.
