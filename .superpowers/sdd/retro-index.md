# Retrospective dataset — classified pain points

Adopted 2026-07-07 (Joel): retrospectives are a DATASET, not meeting
notes. Every pain point a retro surfaces is classified into exactly one
category below and appended to the table. The roadmap is driven by the
empirical distribution — where authors actually struggle — not intuition.

Categories:
- LANG — missing language abstraction
- PASS — missing compiler pass
- SEM — missing semantic model
- TOOL — missing tooling
- MIG — migration debt (transitional; retires with a milestone)
- INH — inherent complexity (real geometry/validation cost any tool pays)

Every future retro MUST append its findings here (one row each) and
update the tally. Controls (places where an existing abstraction made a
cost DISAPPEAR) are logged too — they are the positive evidence.
